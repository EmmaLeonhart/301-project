Assessing categorization consistency across knowledge domains

Analytics Questions

  1. Do Wikipedia categories line up with the Wikidata ontology? — We follow Wikidata's P910 (topic's main category) link from each P31 (instance of) class to its corresponding Wikipedia category and check whether items are actually placed in that category.
  2. Do they align more in certain domains? — We sample five domains (animals, films, cities, chemical elements, albums) and compare how many steps up the Wikipedia category tree are needed to find the P910 (topic's main category) predicted category.
  3. Are some categories of information easier to categorize than others? — We measure depth variance within each domain to identify which areas have consistent vs. inconsistent classification.

Methodology

Data Sources

Wikidata

SPARQL queries against the Wikidata Query Service fetch items that are P31 (instance of) a target class, their P31 (instance of) values, and the P910 (topic's main category) links on those classes.

English Wikipedia

The Wikipedia API retrieves non-hidden categories for each article, plus parent categories up to 5 levels deep.

P910 (topic's main category) Matching

For each item, we follow its P31 (instance of) class's P910 (topic's main category) property to find the Wikidata item for the corresponding Wikipedia category. We then use that item's English Wikipedia sitelink to get the actual category name (e.g. Q7378 → Category:Mammals). If the P31 (instance of) class lacks P910 (topic's main category), we walk up the P279 (subclass of) hierarchy until we find a class that has one.

We then check whether the item is directly in that Wikipedia category. If not, we walk up the Wikipedia parent-category chain and report the minimum depth at which the P910 (topic's main category) predicted category first appears. The "mean depth" shown in the findings is the average of this minimum depth across all items in a domain.

depth = 0  →  item is directly in the P910 (topic's main category) category
depth = 1  →  P910 (topic's main category) category is one level up (parent of a direct category)
depth = N  →  P910 (topic's main category) category is N levels up the Wikipedia parent-category hierarchy

Step 1: Acquire

src/wikidata.py sends SPARQL queries to fetch P31 (instance of) items with English Wikipedia sitelinks, then resolves P910 (topic's main category) links for each P31 (instance of) class (walking up P279 (subclass of) if needed). src/wikipedia.py fetches categories and parent-category chains via the Wikipedia API.

Step 2: Merge & Clean

src/etl.py joins Wikidata P31 (instance of) / P910 (topic's main category) data with Wikipedia categories into a single DataFrame, computing depth to P910 (topic's main category) match for each item.

Step 3: Analyze

src/analysis.py computes per-item overlap ratios and domain-level summaries (mean depth, match rates).

Step 4: Report

reports/generate_report.py reads the CSVs and generates this HTML page and a PDF report. No external rendering tools required.

Key Observations

  1. Chemical elements align directly. The P910 (topic's main category) predicted category appears on every item at depth 0. This domain is small, well-defined, and tightly curated on both systems.
  2. Albums and cities need 2–3 steps. Wikipedia files albums under year/genre subcategories ("2024 pop albums") and cities under geographic subcategories ("Cities in Bavaria") rather than the broad P910 (topic's main category).
  3. Films need about 3 steps. Same pattern as albums — Wikipedia uses fine-grained year/genre categories rather than a single "Films" category.
  4. Animals diverge the most (~5 steps). Wikipedia organizes animals by geography and conservation status ("Mammals of Japan") rather than by biological classification, which is Wikidata's approach.
  5. The pattern: domains where Wikipedia uses broad, type-based categories align directly with Wikidata. Domains where Wikipedia favors specific browsing categories (by year, by country) diverge at the surface but converge higher up the hierarchy.

Answering Our Questions

Q1: Do Wikipedia categories line up with Wikidata ontology?

Sometimes directly, always eventually. Following P910 (topic's main category) links, every domain converges — the question is how many steps up the Wikipedia category tree it takes. Chemical elements match at depth 0; animals need ~5 levels.

Q2: Do they align more in certain domains?

Yes. Small, well-defined domains (chemical elements) align at the surface. Domains with richer Wikipedia categorization schemes (films, animals) require climbing the hierarchy.

Q3: Are some easier to categorize?

Yes. Domains with simple type labels that Wikipedia uses directly are easiest. Domains where Wikipedia prefers geographic, temporal, or genre-based categories over type-based ones show deeper divergence.

Ethics & Limitations

  • Cultural bias: Only English Wikipedia is analyzed, reflecting the interests and knowledge of English-speaking editors.
  • P910 (topic's main category) coverage: The analysis relies on P31 (instance of) classes having P910 (topic's main category) properties. Classes without P910 (topic's main category) are invisible to this comparison.
  • Sample size: The animals domain returned only 3 items due to SPARQL query constraints. Results for this domain are not statistically robust.
  • API rate limits: All requests include rate limiting and a descriptive User-Agent header.
  • Snapshot in time: Both Wikidata and Wikipedia change constantly; results reflect the data at acquisition time (2026-04-05 23:11 UTC).
  • Category hierarchy ceiling: Parent-category traversal is capped at 5 levels. Items that do not match within this window may match at greater depth.
